    Introduction to HTML

    • HTML (HyperText Markup Language) is the foundation of web development.
    • It uses tags (like <p>, <h1>, <img>) to tell browsers how to display content.
    • Every HTML document starts with a DOCTYPE declaration and has two main sections: <head> (metadata)
      and <body> (content).
    • HTML is not a programming language-it’s a markup language that defines structure.

    EXAMPLE :

    • !DOCTYPE html>
    •  <html>
    •   <head>
    •      <title>My First Page</title>
    •   </head>
    •   <body>
    •      <h1>Hello, World!</h1>
    •      <p>This is my first HTML page.</p>
    •   </body>
    • </html>
    Bonus:
    1. Always declare <!DOCTYPE html> for HTML5 compatibility
    2. Use proper indentation for readability
    Additional Info:
    - HTML was first created in 1991 by Tim Berners-Lee.
    - Current version: HTML5 (standard for modern web).
    - Dont't skip heading levels(e.g., <h2> should follow <h1>).

    Example :

    • <!- - - Correct - - ->
    • <h1>Website title</h1>
    • <h2>About us</h2>
      •  
    • <!- - - Wrong - - ->
    • <h1>Website title</h1>
    • <h3>About us</h3>

    Headings and Paragraphs

    • Headings (<h1> – <h6>) represent titles and sub-titles, with <h1> being the most important.
    • Paragraphs (<p>) organize text into readable sections.
    • Browsers automatically add line breaks before and after headings/paragraphs.
    • These elements improve both readability and SEO.

    EXAMPLE :

    • <h1>Main Title</h1>
    • <h2>Subsection</h2>
    • <p>This is a paragraph of text explaining the subsection.</p>
    Bonus:
    1. Use only one <h1> per page (SEO best practice)
    2. Don’t use headings just for making text bigger—use CSS for styling
    Additional Info:
    - Headings help screen readers understand document structure.
    - <p> can include inline tags like <a>, <strong>, <em>.
    - Use <strong> and <em> instead of <b> and <i> for semantic SEO.

    Text Formatting

    • Text formatting tags add meaning or style to text.
    • <b> / <strong> → bold (strong adds importance)
    • <i> / <em> → italics (emphasized)
    • <u> → underline
    • <mark> → highlight

    EXAMPLE :

    • <p> This is <b> bold </b>
    • <i> italic </i>
    • <u> underlined </u>
    • <mark> highlighted </mark> text.</p>
    Bonus:
    1. Prefer <strong> and <em> over <b> and <i> (better for accessibility)
    2. Use <mark> to highlight important points
    Additional Info:
    - Good formatting improves readability and emphasizes meaning.
    - <sup> (superscript) and <sub> (subscript) are used for mathematical notation or chemical formulas.
    Example :
       H 2 O , X 2

    Comments

    • Comments in HTML are written inside <!-- -->.
    • They are ignored by browsers but useful for notes, explanations, or temporarily disabling code.
    • Comments help in collaboration and debugging.

    EXAMPLE :

    • <p>Hello World!</p>
    • <!-- This is a comment, it will not be displayed -->
      •
       
      Bonus:
      1. Use comments to explain code sections
      2. Use comments to disable code without deleting it
      Additional Info:
      - Comments do not affect page speed.
      - Avoid overusing comments—write clean code instead.

    Working with Basic Elements :

    Links

    • Links connect one webpage to another.
    • The <a> tag is used with the href attribute to specify the destination.
    • Links can open in the same tab or new tab using the target attribute.
    • They can also link to sections of the same page, email addresses, or files.

    EXAMPLE :

    • <a href="https://www.google.com" target="_blank">Visit Google</a>
    Bonus:
    1.Use target="_blank" for external websites so users don’t leave your site
    2.Add the title attribute for accessibility and better SEO.
    Additional Info:
    - HTML was first created in 1991 by Tim Berners-Lee.
    - Current version: HTML5 (standard for modern web).
    - Dont't skip heading levels(e.g., <h2> should follow <h1>).

    Example :

    • <a href="about.html" title="Go to About Page">About Us</a>

    Images

    • Images are added using the <img> tag.
    • The src attribute specifies the path of the image.
    • The alt attribute provides text if the image fails to load (important for accessibility and SEO).
    • You can control size with width and height.

    EXAMPLE :

    • <img src="logo.png" alt="Website Logo" width="200">
      •
       
      Bonus:
      1.Always include alt text.
      2.Use relative paths for local images and absolute URLs for external images.
      3. Use width="100%" for responsive banners.
      Additional Info:
      - Common formats: JPEG (photos), PNG (transparent backgrounds), GIF (animations), SVG (scalable vector graphics).
      - Images can also be made responsive with max-width: 100% in CSS.

    Lists

    • Lists help organize content.
    • Lists can be nested (list inside another list).
    • HTML supports:
      • <ul> (unordered list — bullet points)
      • <ol> (ordered list — numbers)
      • <dl> (description list — terms with details)

    EXAMPLE :

    • <ul>
    • <li>HTML</li>
    • <li>CSS</li>
    • </ul>
  • List Attributes:
    • start → Defines the starting number in <ol>.
    • reversed → Makes an ordered list count backwards.
    • type → Defines numbering style (1, A, a, I, i).

    EXAMPLE :

    • <ol type="I" start="5" reversed>
    • <li>Point One</li>
    • <li>Point Two</li>
    • </o>
    Bonus:
    1.Use <ol> for steps or sequences.
    2. Use lists for navigation menus, steps in a process, or grouped data for better readability.
    Additional Info:
  • CSS can customize list markers (symbols, numbers, icons, or even images).

    • Tables

    • Tables are used for structured data
    • <table> creates the table, <tr> defines rows, <th> defines headers, <td> defines data cells.
    • Attributes like border can be used, but CSS styling is recommended.

    EXAMPLE :

    • <table border="1">
    • <tr>
    •  <th>Name</th>
    •  <th>Age</th>
    • </tr>
    • <tr>
    •   <td>Mohan</td>
    •  <td>25</td>
    • </tr>
    • </table>
      •
       
      Bonus:
      1.Use <caption> to give titles to tables.
      2.Apply CSS for professional styling (borders, colors, spacing).
      Additional Info:
      - Avoid using tables for page layout; use them only for   tabular data.
      - For accessibility, always use table headers and captions   (<caption>).
      - Use <thead>, <tbody>, and <tfoot> for better structure and     readability.
      - Attributes like colspan and rowspan merge multiple cells.

      Example :

      • < table>
      • < caption > Student Data < /caption >
      • <tr>
      •    <th> Name </th>
      •   < th> Age </th>
      • </tr>
      • <tr>
      •    <td> Mohan </td>
      •    <td> 21 </td>
      • < /tr >
      • < /table >
